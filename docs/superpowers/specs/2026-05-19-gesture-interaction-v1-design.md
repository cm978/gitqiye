# 基于 ResNet50-LSTM 的动态手势识别与虚拟交互系统 - 第一版设计

## 目标

构建一个基于 Flask、PyTorch 和 Web 前端的动态手势识别与虚拟交互系统。系统以 ResNet50-LSTM 模型识别连续视频帧中的动态手势，并将识别结果用于网页控制、媒体控制和粒子交互。

第一版聚焦可运行、可训练、可展示，不包含用户登录、数据库、多用户管理和在线训练。

## 核心识别类别

模型输出 8 类：

| 类别 | 说明 |
| --- | --- |
| no_gesture | 无有效手势 |
| swipe_left | 左滑 |
| swipe_right | 右滑 |
| swipe_up | 上滑 |
| swipe_down | 下滑 |
| click | 点击 / 确认 |
| zoom_in | 放大 / 扩散 |
| zoom_out | 缩小 / 聚拢 |

对外展示为 7 种核心动态交互手势，并具备无手势检测能力。

## 功能模块

### 实时识别中心

- 打开和关闭摄像头。
- 显示实时摄像头画面。
- 显示当前识别手势、置信度和触发动作。
- 显示当前交互模式。
- 显示最近识别历史。
- 显示模型状态。

### 网页控制模式

- 左滑：上一页 / 上一张。
- 右滑：下一页 / 下一张。
- 上滑：页面上滚。
- 下滑：页面下滚。
- 点击：确认 / 进入。

### 媒体控制模式

- 左滑：上一首。
- 右滑：下一首。
- 上滑：音量加。
- 下滑：音量减。
- 点击：播放 / 暂停。

### 粒子交互模式

- 左滑 / 右滑：粒子按方向流动。
- 上滑 / 下滑：粒子整体方向改变。
- 点击：切换颜色或爆发效果。
- 放大：粒子扩散。
- 缩小：粒子聚拢。

### 手势指令映射管理

- 展示每个手势在当前模式下对应的操作。
- 第一版支持查看和切换预设模式。
- 暂不做复杂自定义保存。

### 识别历史记录

- 记录最近识别结果。
- 字段包括时间、手势、置信度、模式、执行动作。
- 第一版保存在浏览器内存或后端内存，不接数据库。

### 模型状态监控

- 展示模型名称、输入帧数、推理 FPS、置信度阈值、防抖状态、运行设备。
- 支持防抖机制：置信度超过阈值且稳定出现时才触发动作。

### 自定义数据采集

- 选择手势类别。
- 倒计时后录制 2-3 秒视频。
- 保存到对应类别目录。
- 显示每类样本数量。
- 第一版只采集数据，不在网页内执行训练。

## 模型方案

采用 ResNet50-LSTM：

1. 前端或离线脚本采集连续视频帧。
2. 每段样本采样 16 帧。
3. 每帧缩放为 224x224。
4. 使用 ImageNet 预训练 ResNet-50 提取 2048 维图像特征。
5. 使用 LSTM 学习时间变化。
6. 全连接层输出 8 类手势概率。

推荐默认参数：

- sequence_length: 16
- image_size: 224
- hidden_size: 256
- num_layers: 1
- dropout: 0.3
- loss: CrossEntropyLoss
- optimizer: Adam
- learning_rate: 1e-4
- training_device: cuda by default; CPU is only for tests and small smoke checks

## 数据方案

第一阶段使用 IPN Hand 数据集筛选类别：

| IPN Hand 类别 | 项目类别 |
| --- | --- |
| Non-gesture | no_gesture |
| Throw left | swipe_left |
| Throw right | swipe_right |
| Throw up | swipe_up |
| Throw down | swipe_down |
| Click with one finger | click |
| Zoom in | zoom_in |
| Zoom out | zoom_out |

第二阶段使用网页采集模块录制少量本地样本，每类建议 30-80 段，用于后续微调。

## 系统架构

前端负责摄像头采集、页面交互、媒体控制、粒子动画和结果展示。

Flask 后端负责页面服务、推理接口、数据采集保存、模型状态接口。

PyTorch 模型负责视频帧序列分类。模型加载后常驻内存，推理接口复用同一个模型实例。

训练脚本默认使用 GPU。测试、网页 demo 和本地推理可以在 CPU 上运行。

## 接口草案

| 接口 | 方法 | 功能 |
| --- | --- | --- |
| / | GET | 主页面 |
| /api/predict | POST | 接收帧序列并返回手势结果 |
| /api/status | GET | 返回模型状态 |
| /api/collect | POST | 保存自定义采集样本 |
| /api/sample-counts | GET | 返回每类样本数量 |

## 开发范围

第一版实现工程结构、基础页面、推理流程、模型定义、训练脚本、数据集读取、自定义采集接口和前端三种交互模式。

如果没有训练好的权重，系统应支持 demo 模式：后端返回可控的模拟识别结果，前端交互可以先完整跑通。模型训练完成后替换为真实推理。

## 非目标

- 不做用户登录。
- 不做数据库。
- 不做在线训练。
- 不做复杂权限管理。
- 不做移动端深度适配。
